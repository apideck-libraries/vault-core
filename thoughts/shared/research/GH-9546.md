# Research: vault-core — GH-9546 vault-core Sync

## Research Question

vault-core (`@apideck/react-vault`) was partially synchronized with the GH-9546 security fixes. The CSRF/nonce infrastructure is correctly in place, but two gaps remain:

1. The authorize POST body omits `redirect_uri` when `session.redirect_uri` is not set — unify's `SecureAuthorizeConnectionRequest` schema requires it with `minLength: 1`, causing HTTP 400.
2. The postMessage `messageHandler` has no `event.origin` check — deferred from the main GH-9546 cycle, to be addressed now.

Fix both gaps in `src/utils/connectionActions.ts` and update tests in `test/connectionActions.test.ts`.

## Summary

vault-core correctly implements nonce-based CSRF protection, the authorize → popup → postMessage → confirm flow, and TypeScript types for all API shapes. The two remaining gaps are both localized to `connectionActions.ts`.

Gap 1 is breaking: unify requires `redirect_uri` always, but vault-core only sends it when `session?.redirect_uri` is truthy. The fallback value already exists as `REDIRECT_URL = 'https://vault.apideck.com/oauth/callback'` in `src/constants/urls.ts` — it just isn't imported in `connectionActions.ts`.

Gap 2 is a security hardening: the `messageHandler` should reject messages from origins other than the domain of the `redirect_uri` used for the authorize call. With the Gap 1 fix in place, the trusted origin is always derivable from `session?.redirect_uri ?? REDIRECT_URL`.

## Implementation Details

### Gap 1 — Missing `redirect_uri` fallback (BREAKING)

**Location**: `src/utils/connectionActions.ts:198–201`

Current code:
```ts
const authorizeBody: Record<string, string> = { nonce };
if (session?.redirect_uri) {
  authorizeBody.redirect_uri = session.redirect_uri;
}
```

Required change:
```ts
const authorizeBody: Record<string, string> = {
  nonce,
  redirect_uri: session?.redirect_uri ?? REDIRECT_URL,
};
```

Also add the import at the top of `connectionActions.ts`:
```ts
import { REDIRECT_URL } from '../constants/urls'
```

(`BASE_URL` is already imported from that file — check the existing import and add `REDIRECT_URL` to it.)

### Gap 2 — Missing `event.origin` validation (Security)

**Location**: `src/utils/connectionActions.ts:257`

Current code:
```ts
const messageHandler = async (event: MessageEvent) => {
  const { data } = event;
  // No event.origin check
  if (data?.type === 'oauth_complete' && data?.serviceId === serviceId) {
```

Required change — add origin check as the first thing in the handler:
```ts
const trustedOrigin = new URL(session?.redirect_uri ?? REDIRECT_URL).origin;
const messageHandler = async (event: MessageEvent) => {
  if (event.origin !== trustedOrigin) return;
  const { data } = event;
  if (data?.type === 'oauth_complete' && data?.serviceId === serviceId) {
```

`trustedOrigin` is derived once before the handler is defined, using the same `session?.redirect_uri ?? REDIRECT_URL` expression used for `redirect_uri` in the authorize body. This keeps the two values in sync.

- Default: `new URL('https://vault.apideck.com/oauth/callback').origin` → `'https://vault.apideck.com'`
- Custom session redirect: `new URL('https://custom.example.com/callback').origin` → `'https://custom.example.com'`

### Full authorize flow after fix

1. `generateAndStoreNonce(serviceId)` → stores `vault_oauth_nonce_{serviceId}` in sessionStorage
2. `POST {connectionsUrl}/{unifiedApi}/{serviceId}/authorize` with `{ nonce, redirect_uri: session?.redirect_uri ?? REDIRECT_URL }` → reads `data.authorize_url`
3. `window.open(authorize_url, '_blank', windowFeatures)` → OAuth popup
4. Compute `trustedOrigin = new URL(session?.redirect_uri ?? REDIRECT_URL).origin`
5. `window.addEventListener('message', messageHandler)` — handler rejects `event.origin !== trustedOrigin`
6. On `oauth_complete` from trusted origin: `verifyAndClearNonce` → `POST /confirm` with `{ confirm_token }`
7. After confirm: `mutate` SWR cache → `doCleanup()`

## Plan Decisions

- **[vault-core] redirect_uri fallback**: Use `session?.redirect_uri ?? REDIRECT_URL` — `REDIRECT_URL` is already exported from `src/constants/urls.ts` as `'https://vault.apideck.com/oauth/callback'`. Import it in `connectionActions.ts` alongside existing imports from that module.
- **[vault-core] event.origin validation**: Derive trusted origin from `new URL(session?.redirect_uri ?? REDIRECT_URL).origin` at the point where `messageHandler` is defined (i.e., inside `handleAuthorize` where `session` is in scope). - **[vault-core] test updates**: Extend existing tests in `test/connectionActions.test.ts`. Do not create new test files. Follow existing mock patterns for `window.addEventListener('message', ...)` and `window.open`.
